# 부록 3. MVP 외 확장 설계 - 결제수단 기반 충전

## 3-1. 범위

> 본 문서는 현재 MVP 본 구현 범위에서 제외된 결제수단 기반 충전 확장 초안을 정리한다.

- 등록된 카드/계좌 토큰 기반 충전
- 외부 결제 참조 ID 기록
- 충전 결과와 외부 결제 트랜잭션 연결

## 3-2. 페이머니 충전 API 확장

| 항목 | 내용 |
|------|------|
| 목적 | 등록된 카드/계좌로 페이머니(지갑) 충전 |
| 처리 방식 | 결제수단 검증 → 외부 결제 → 페이머니 잔액 증가 → 원장 기록 |
| 주요 엔드포인트 | POST /api/v1/wallets/{walletId}/charge |
| 관련 FR | FR-004 (기존) + FR-CH01~CH03 (신규) |

## 3-3. 고려 사항

- 외부 결제 성공 시에만 잔액 증가
- `LedgerEntry.externalRefId`에 외부 결제 트랜잭션 ID 기록
- 결제수단 상태가 `DELETED`이면 충전 불가
- 현재 단순 충전 유즈케이스와 구분해 별도 결제수단 검증 계층 필요
